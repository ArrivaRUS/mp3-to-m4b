# mp3-to-m4b — бренд-базис (финал)

> Принято на ⛔ G2a: направление **«Книга + play»** · **единое семейство** с `fb2-to-epub` ·
> фирменный цвет cyan `#34E0D2` → teal `#22B5E0` → indigo `#4A6BFF`.
> Это **источник значений для дизайн-токенов (P9)** и макетов (UXAnalyst). Все значения —
> конкретные. Концепты-история: `branding/concepts/`.

---

## 1. Знак (логотип)

**Идея:** раскрытая книга-лист (общий жест семьи fb2) + **play-треугольник** на корешке =
аудиокнига одним знаком. Тёмная панель, мягкое свечение-bloom, drop-shadow — грамматика семьи.

**Файлы:**
| Файл | Назначение |
|---|---|
| `branding/logo.svg` | Горизонтальный lockup (знак-тайл + вордмарк `mp3-to-m4b`). Основной логотип. |
| `branding/icon-app.svg` | Мастер app-иконка, full-bleed 1024×1024. Использовать при **≥64 px**. |
| `branding/icon-small.svg` | Упрощённая иконка (без линий страниц, крупнее book+play) для **16–48 px** (Dock-маленький, фавикон). |
| `branding/icon-mono.svg` | Одноцветный знак (negative-space play), `fill`/`currentColor`. Для ч-б / печати / мелких контекстов. |
| `branding/icon-menubar-template.svg` | macOS **template image** 22×22 (чёрный силуэт + сквозные вырезы), тинтуется системой. |

**Правило app-иконки:** рисуется **full-bleed** на 1024×1024 со сплошной тёмной заливкой углов
(`#070B10`) — **скругление делает система** (macOS/Tahoe). Не вшивать squircle-клип в мастер.

## 4. Типографика — питает токены `font.*`

| Токен | Значение |
|---|---|
| `font.family.display` | `-apple-system, BlinkMacSystemFont, "SF Pro Display", "Segoe UI", Helvetica, Arial, sans-serif` |
| `font.family.text` | `-apple-system, BlinkMacSystemFont, "SF Pro Text", "Segoe UI", Helvetica, Arial, sans-serif` |
| `font.weight.bold` | `700` (вордмарк, заголовки) |
| `font.weight.semibold` | `600` (caps-микроподпись, акценты) |
| `font.weight.regular` | `400` (текст интерфейса) |
| `font.tracking.wordmark` | `-2` (крупный вордмарк / заголовки) |
| `font.tracking.caps` | `+6` (caps-микроподпись) |

**Характер:** «нативный macOS», чистый, тех-аккуратный. Вордмарк двухцветный: `mp3` —
`color.text.high`, стрелка `→` и `m4b` — фирменный градиент (1-в-1 принцип соседа).

**Кириллица (важно для запасных обложек, не для логотипа):** логотип — латиница. Для
SVG→PNG-обложек (F6, движок cairosvg) использовать только «зелёный» список из
`research/m4b-toolchain.md` §7 — **Georgia / Helvetica / Helvetica Neue / Gill Sans / Palatino /
Hoefler Text / Baskerville-roman**; курсив-антиква — **Georgia italic** (не Baskerville italic);
**не** Didot/Futura/Avenir; всегда ставить серверный fallback-стек. Это часть единого бренд-стека,
но к самому знаку отношения не имеет.

## 5. Правила использования знака

**Можно:**
- Логотип целиком — `logo.svg` (знак + вордмарк) на тёмном фоне.
- Только знак — `icon-app.svg` (≥64 px) / `icon-small.svg` (16–48 px) для приложения/Dock/фавикона.
- Моно/template — `icon-mono.svg` (ч-б, перекраска одним цветом) / `icon-menubar-template.svg` (статус-бар).
- Перекраска **только моно-версии** одним цветом из палитры или нейтралью; цветной знак —
  только в фирменном градиенте.

**Нельзя:**
- Менять цвета фирменного градиента, разворачивать/наклонять знак, добавлять обводку/тень
  поверх готового знака, ставить цветной знак на светлый «шумный» фон без тёмной подложки.
- Растягивать непропорционально; перерисовывать play/книгу; вшивать squircle-клип в мастер-иконку.
- Набирать вордмарк другим шрифтом/начертанием (только SF Pro Bold, tracking −2).

**Минимальные размеры:**
- `logo.svg` (lockup): не меньше **160 px** по ширине (ниже — микроподпись нечитаема; брать только знак).
- Знак цветной: `icon-app` — **≥64 px**; **<64 px → `icon-small`**; статус-бар → `icon-menubar-template` (22 px).
- Моно: читаем до **22 px**.

## 6. Clear-space и отступы

- **Clear-space знака:** свободное поле вокруг знака = **¼ высоты знака** со всех сторон
  (ничего постороннего в этой зоне).
- **Lockup внутренние отступы (как в `logo.svg`, 880×220 канва):** тайл знака 128×128,
  поле слева 36; вордмарк `mp3` baseline x≈196; зазор между `mp3`→стрелка→`m4b` ~ 48 px от
  глифа до глифа; микроподпись под вордмарком, baseline +40 от базовой линии слов.
- **Радиус тайла знака в lockup:** `30` на канве 220 (≈ 13.6% — согласуется со squircle-ощущением).
- **Скругление app-иконки:** **не задаём** (система). Для превью-карточек/не-нативных контекстов
  использовать `border-radius ≈ 22%` от стороны.
- **Тёмная подложка-диск под play:** радиус ≈ 0.10 стороны иконки, `color.bg.tile` @ opacity
  0.55–0.60 — обязательна, чтобы play не «слипался» со страницами.

## 7. Семейная связь с fb2-to-epub (зафиксировано)

| Элемент | fb2-to-epub | mp3-to-m4b |
|---|---|---|
| Тёмная панель + bloom + drop-shadow + full-bleed app-иконка | да | **да (то же)** |
| Вордмарк SF Pro Bold, tracking −2, двухтон | `fb2` / `epub` | **`mp3` / `m4b`** |
| Морф-стрелка между частями имени | `→` со «спарком» | **`→`** (спарк заменён на play в знаке) |
| Фирменный градиент | тёплый orange→red→magenta | **холодный cyan→teal→indigo** |
| Мотив знака | страница + спарк | **книга + play** (аудиокнига) |

Две иконки читаются как родственники (общий каркас) и **различаются в Dock** (тёплый vs
холодный, спарк vs play).

## 8. Что изменено против концепта 02 (доводка)

1. **Убрана «бровь».** Старый highlight-path (дуга-полумесяц вверху) → заменён прямым
   `top-edge` linear-sheen (opacity 0.10→0, высота ~320). Крупный размер чистый, без арки.
2. **Play разведён со страницами.** (а) Линии правой страницы укорочены и сдвинуты к внешнему
   краю; (б) добавлена тёмная подложка-диск под play (`#0B1118` @0.55); (в) свечение play —
   tight-glow (stdDeviation 22 → 8), больше не размывается на страницы; (г) play чуть уменьшен и
   отцентрован на корешке; добавлен тонкий шов корешка.
3. **Глубина разворота:** правая страница — отдельный, чуть более глубокий градиент
   (`gradient.brand.deep`), чтобы разворот читался объёмнее.
4. **Добавлены пак-файлы:** mono (negative-space), menubar-template 22 px, icon-small (16–48 px).

## 9. Проверка рендером (выполнена)

Все финальные SVG провалидированы (`xmllint` OK) и проверены **SVG→PNG (headless Chrome)**:
- `icon-app` — 256/48/24 px на тёмном и светлом фоне (book+play читается; «бровь» исчезла;
  play отделён от страниц).
- `icon-small` vs `icon-app` @24 px — small заметно чётче (обоснование отдельного файла).
- `icon-mono` — ink-on-light и white-on-dark, 120/40/22 px (negative-space play читается).
- `icon-menubar-template` — black-on-light и white-on-dark (тинт), 44/22/16 px (силуэт держится).
- `logo` — вордмарк SF Pro подхватился, двухтон и стрелка корректны.

Только **сплошные заливки**; тонких штрихов под фильтрами нет (уроки `cairosvg-thin-stroke` /
research §7). PNG-превью доводки — во временной папке сессии (не коммитятся); пере-рендерить
из SVG можно в любой момент.
