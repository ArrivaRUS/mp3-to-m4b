# User Stories — mp3-to-m4b

> Версия: 1.0 · Дата: 2026-06-28 · Автор: ProductManager (студия Юрки)
> Источник: `prd/PRD.md`. Формат: **As a {роль}, I want {действие}, so that {ценность}**.
> Роль одна: **Антон** (power user macOS, делает аудиокниги для себя).
> Каждая история: INVEST-проверка · acceptance (Given-When-Then) · привязка к фиче PRD · приоритет.
> Оценка размера: **S** (≤0.5 дня), **M** (≈1 день), **L** (≈2–3 дня) — для беклога.

---

## Epic'и (крупные куски ценности)

| Epic | Ценность | Фичи PRD |
|---|---|---|
| **E1. Установка и слежение** | Поставить, выбрать папку, агент следит | F1, F12, F13 |
| **E2. Обнаружение и распознавание** | Найти книги в папке, понять автор/название/главы | F2, F3 |
| **E3. Подтверждение перед сборкой (ядро)** | Проверить и подтвердить качество+обложку до сборки | F4, F5, F9 |

| **E6. Нарезка на части** | Опц. порезать большую книгу по границам глав | F8 |
| **E7. Надёжность (edge/идемпотентность/отмена)** | Не ломаться на битых входах, не плодить мусор | F10, F11 |

## E1. Установка и слежение

### US-1.1 — Проверка ffmpeg при установке — P0 · S
**As Антон, I want** при первом запуске узнать, есть ли ffmpeg, **so that** не получить
молчаливые сбои сборки.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given ffmpeg/ffprobe нет, When запуск, Then экран с `brew install ffmpeg` +
  «Проверить снова», установка не идёт дальше до появления ffmpeg.
- Фича: F1.

### US-1.2 — Выбор отслеживаемой папки и установка агента — P0 · M
**As Антон, I want** выбрать папку и поставить фоновый агент, **so that** дальше просто кидать
в неё mp3.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given ffmpeg есть, When подтвердил дефолт `~/Desktop/mp3-to-m4b`, Then папка
  создана, агент поставлен идемпотентно (повтор не плодит агентов), показан Status.
- Фича: F1.

### US-1.3 — Подсказка Full Disk Access для защищённой папки — P1 · S
**As Антон, I want** понятную инструкцию по FDA, если папка в Desktop/Documents/Downloads,
**so that** агент мог читать файлы и сборка не «молчала».
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given папка в защищённой зоне, When установка завершилась, Then показан точный путь
  раннера и шаги выдачи FDA.
- Фича: F1.

### US-1.4 — Экран статуса со слежением и прогрессом — P1 · M
**As Антон, I want** видеть папку, состояние агента и прогресс пачки, **so that** понимать, что
всё работает, и быстро открыть результаты.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given идёт сборка, When открыл приложение, Then Status показывает папку, «агент вкл»,
  кольцо-прогресс `M/N`, последние книги; обновляется live без ручного refresh.
- Фича: F12.

### US-1.5 — Смена отслеживаемой папки — P2 · S
**As Антон, I want** сменить папку позже, **so that** перенаправить слежение без переустановки.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given приложение установлено, When выбрал другую папку, Then агент следит за новой
  (переустановка идемпотентна), старые результаты не трогаются.
- Фича: F12/F1.

### US-1.6 — Настройки: дефолты, FDA, обновление, сброс статистики — P2 · M
**As Антон, I want** задать дефолтные параметры и обслужить приложение, **so that** не
настраивать каждую книгу и держать приложение свежим.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given изменил дефолтный битрейт на 128k, When появилась новая книга, Then окно
  подтверждения предзаполнено 128k.
- Фича: F13.

## E2. Обнаружение и распознавание

### US-2.1 — Подпапка с mp3 → одна книга — P0 · M
**As Антон, I want** чтобы подпапка-сборник распозналась как одна книга, **so that** не
раскладывать файлы вручную.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given подпапка `Автор - Книга/` с `01.mp3…10.mp3`, When копирование завершено, Then
  одна книга `detected`, главы упорядочены натурально (01..10, не 1,10,2).
- Фича: F2. Решение: D1, D5.

### US-2.2 — Диалог группировки одиночных mp3 в корне — P0 · M
**As Антон, I want** выбрать «объединить / по отдельности» для разрозненных mp3 в корне,
**so that** сам решать, книга это или отдельные файлы.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given 3 одиночных mp3 в корне, When агент сработал, Then диалог D1; When «по
  отдельности» → 3 книги; When «объединить» → 1 книга из 3 глав.
- Фича: F2. Решение: D1.

### US-2.3 — Игнор не-mp3 мусора и недокопированных файлов — P0 · S
**As Антон, I want** чтобы мусор и недокопированные файлы не попадали в книгу, **so that** не
получить битую сборку.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given в папке mp3 + `.txt`/`.DS_Store`/частичный `.mp3.download`, When отбор глав,
  Then в книгу попадают только дописанные mp3; недокопированный не захватывается, пока размер
  не стабилизировался.
- Фича: F2. Edge: E9, E10.

### US-2.4 — Автор/название из тегов, иначе из имени папки — P0 · M
**As Антон, I want** чтобы автор/название определялись из тегов или имени папки, **so that** не
вбивать руками.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given теги пустые, папка «Толстой - Война и мир», When распознавание, Then
  автор=«Толстой», название=«Война и мир»; Given теги заполнены, Then берём `album`/`artist`.
- Фича: F3. Edge: E1.

### US-2.5 — Имена и порядок глав с fallback — P0 · S
**As Антон, I want** имена глав из `title`, иначе из имени файла без номера, **so that** главы
читались по-человечески.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given часть файлов без `title`, When распознавание, Then имя главы — из имени файла
  без ведущего номера (`^\d+[\s._-]+`); порядок ← `track`, иначе натуральная сортировка;
  кириллица корректна.
- Фича: F3. Решение: D5.

## E3. Подтверждение перед сборкой (ядро)

### US-3.1 — Всплытие окна подтверждения при обнаружении книги — P0 · L
**As Антон, I want** чтобы окно само всплывало с распознанной книгой, **so that** не пропустить
момент подтверждения.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given книга → `pending-confirm`, When это произошло, Then окно всплывает поверх (а
  если закрыто — открывается) с автор/название/главами/обложкой/параметрами/предпросмотром.
- Фича: F4, F9. Решение: D4 (ядро).

### US-3.2 — Без «ок» сборка не стартует (инвариант I2) — P0 · M
**As Антон, I want** чтобы без моего подтверждения ничего не собиралось, **so that** не получить
книгу с неверными параметрами, пока меня нет.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given окно открыто и я ничего не нажал, When проходит время, Then сборка НЕ стартует,
  книга остаётся `pending-confirm` в очереди.
- Фича: F4. Решение: D4, I2.

### US-3.3 — Правка распознанных полей и старт сборки — P0 · M
**As Антон, I want** поправить автор/название/имена глав и нажать «Собрать», **so that**
исправить кривые теги до сборки.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given поправил название и битрейт, When «Собрать», Then книга → `converting` с
  применёнными параметрами; Given название пустое, When «Собрать», Then кнопка заблокирована
  с подсказкой.
- Фича: F4. Edge: валидация.

### US-3.4 — Выбор параметров качества — P0 · M
**As Антон, I want** задать битрейт/каналы/SR перед сборкой, **so that** баланс качество/размер
под мою книгу.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given не менял параметры, When сборка, Then дефолт AAC 192k стерео 44100; Given
  разные SR/каналы входов, When сборка, Then приведены к целевым без дрейфа границ.
- Фича: F5. Решение: D2. Edge: E2.

### US-3.5 — Очередь из нескольких книг — P0 · M
**As Антон, I want** видеть очередь ожидающих подтверждения книг, **so that** обработать пачку
из нескольких папок.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given скопированы 5 папок разом, When агент сработал, Then 5 книг в `pending-confirm`;
  When подтверждаю по одной, Then каждая идёт в `converting` независимо, прогресс `M/5`.
- Фича: F9, F4.

### US-3.6 — Прогресс сборки (пачка + книга) — P0 · S
**As Антон, I want** видеть прогресс сборки, **so that** понимать, сколько осталось.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given идёт сборка пачки, When смотрю Status, Then кольцо-прогресс показывает `M/N`
  книг и продвигается по главам текущей книги.
- Фича: F9.

### US-3.7 — Отложить книгу / «применить ко всем» — P1 · M
**As Антон, I want** отложить книгу в очередь и применить параметры ко всем ожидающим,
**so that** быстро прогнать пачку с общим пресетом.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given несколько книг в очереди, When «Применить параметры ко всем ожидающим», Then
  всем задан общий пресет/нарезка (обложку всё равно подтверждаю по каждой); When «Позже» →
  книга остаётся `pending-confirm`.
- Фича: F4/F9.

## E4. Обложка

### US-4.1 — Обложка из встроенной в mp3 — P0 · S
**As Антон, I want** использовать обложку из mp3, если она есть, **so that** не искать то, что
уже вшито.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given в одном mp3 есть APIC, When распознавание, Then обложка берётся из первого
  такого файла (`from-mp3`), сеть не опрашивается.
- Фича: F6. research §4.

### US-4.2 — Поиск обложки в сети (квадрат 1:1) — P0 · L
**As Антон, I want** чтобы при отсутствии обложки в mp3 шёл поиск в сети, **so that** взять
настоящую обложку аудиокниги.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given в mp3 обложки нет и есть интернет, When поиск, Then показываются квадратные
  (≈1:1) кандидаты в пейджере; портретные/непохожие отфильтрованы.
- Фича: F6. research §6.

### US-4.3 — Генерация квадратных вариантов при отсутствии обложки — P0 · L
**As Антон, I want** чтобы при отсутствии и в mp3, и в сети сгенерировались варианты,
**so that** ни одна книга не осталась без обложки.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given обложки нет и интернета нет, When цепочка обложки, Then генерируются квадратные
  SVG→PNG варианты; книга не остаётся без обложки (G4).
- Фича: F6. research §7. (Шаблоны рисует BrandDesigner/UXAnalyst на дизайне.)

### US-4.4 — Выбор/переиск/ручная замена обложки — P0 · M
**As Антон, I want** перелистывать кандидатов, искать ещё по своему запросу и заменять обложку
своим файлом, **so that** получить именно ту обложку, что хочу.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given кандидаты показаны, When «искать ещё» со своим запросом, Then показываются новые
  (исключая уже виденные); When перетащил свой jpg, Then обложка заменена (кроп к квадрату),
  под-состояние `user-replaced`.
- Фича: F6.

## E5. Сборка m4b

### US-5.1 — Сборка валидного .m4b с главами и обложкой — P0 · L
**As Антон, I want** получить валидный `.m4b` с главами и обложкой, **so that** он открывался
как аудиокнига в Apple Books и плеерах.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given подтверждённая книга из N mp3, When сборка завершилась, Then рядом один `.m4b`
  (brand `M4A`, faststart) с N главами (правильные START/END, кликабельны), обложкой и
  метаданными автор/название; исходные mp3 не изменены (I1).
- Фича: F7. research §1.

### US-5.2 — Атомарная сборка (нет полу-файла при сбое) — P0 · S
**As Антон, I want** чтобы при сбое не оставался полу-`.m4b`, **so that** не путаться в битых
результатах.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given сборка упала/прервалась, When это произошло, Then финального `.m4b` нет (только
  temp, который удалён).
- Фича: F7. Edge: E18.

### US-5.3 — Результат рядом с источником, исходники целы — P0 · S
**As Антон, I want** чтобы `.m4b` появлялся рядом, не трогая mp3, **so that** мои исходники
оставались нетронутыми.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given сборка завершилась, When смотрю папку, Then рядом `.m4b`, все исходные mp3
  не изменены и не удалены (I1).
- Фича: F7. Инвариант I1.

## E6. Нарезка на части

### US-6.1 — Сборка одним файлом (дефолт) — P0 · S
**As Антон, I want** по умолчанию получать книгу одним файлом, **so that** не плодить части без
необходимости.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given нарезка выключена (дефолт), When сборка, Then результат — один `.m4b`.
- Фича: F8. Решение: D6 (дефолт — одним файлом).

### US-6.2 — Нарезка по порогу по границам глав — P1 · L
**As Антон, I want** опционально резать книгу по порогу размера по границам глав, **so that**
управлять большой книгой по частям.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given нарезка включена с порогом ~300 МБ и книга крупнее, When сборка, Then M частей,
  каждая валидный `.m4b` со своей обложкой, `title=«…, Часть N из M»`, `track=N/M`, общий album;
  ни одна глава не разрезана.
- Фича: F8. research §3. Решение: D6.

### US-6.3 — Предпросмотр размера и числа частей — P1 · M
**As Антон, I want** видеть оценку «≈ N частей по ~X МБ» при выборе порога, **so that** выбрать
порог осознанно до сборки.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given окно подтверждения, When меняю порог, Then обновляется предпросмотр числа
  частей и размера; Given порог < одной главы, Then показано, что части будут крупнее порога.
- Фича: F8. Edge: E17.

## E7. Надёжность (edge / идемпотентность / отмена)

### US-7.1 — Идемпотентность (не пересобирать свежее) — P0 · M
**As Антон, I want** чтобы повторное срабатывание не пересобирало свежую книгу, **so that**
кидать пачки спокойно, без лишней работы.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given рядом есть `.m4b` новее всех исходных mp3, When агент снова сработал, Then книга
  не пересобирается (тихо пропущена); Given изменил/добавил mp3, Then пересборка после
  подтверждения.
- Фича: F10. Edge: E6. (G5.)

### US-7.2 — Обработка битого/нечитаемого mp3 — P0 · M
**As Антон, I want** понятную ошибку при битом mp3, **so that** решить — пропустить файл или
отменить книгу.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given один mp3 битый, When распознавание/сборка, Then книга → `error` с указанием
  файла и выбором «пропустить файл и собрать остальное / отменить».
- Фича: F10. Edge: E3.

### US-7.3 — Обработка нехватки места на диске — P0 · S
**As Антон, I want** понятную ошибку при нехватке места, **so that** освободить место и
повторить без мусора.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given места не хватает, When сборка, Then книга → `error` «не хватает места»,
  частичный файл удалён, исходники целы.
- Фича: F10. Edge: E5.

### US-7.4 — Корректная деградация без интернета (обложка) — P0 · S
**As Антон, I want** чтобы без интернета цепочка обложки не вставала, **so that** книга всё
равно собралась с обложкой.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given интернета нет и в mp3 обложки нет, When цепочка обложки, Then веб-поиск
  пропущен, сразу генерация вариантов; книга собирается с обложкой.
- Фича: F6/F10. Edge: E4. (G4.)

### US-7.5 — Отмена идущей сборки — P1 · M
**As Антон, I want** прервать идущую сборку, **so that** не ждать ненужную/ошибочную книгу.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given книга в `converting`, When «Отмена», Then ffmpeg остановлен, temp удалён,
  исходники целы, статус → `cancelled`, книгу можно собрать заново.
- Фича: F11. Edge: E7.

### US-7.6 — Очень длинная книга / сотни файлов — P2 · M
**As Антон, I want** чтобы очень длинная книга собиралась без зависания, **so that** большие
сборники тоже работали.
- **INVEST:** I✔ N✔ V✔ E✔ S✔ T✔.
- **AC:** Given десятки часов / сотни файлов, When сборка, Then прогресс по главам не зависает;
  при упоре в лимит открытых файлов — fallback на demuxer с предварительным приведением входов.
- Фича: F10. Edge: E8. research §1a note.

## INVEST-замечания

Все истории прошли INVEST. Две требовали явной нарезки и переразбиты ещё на этапе формулировки:

1. **«Окно подтверждения» (исходно одна большая)** → разбита на независимые: всплытие (US-3.1),
   инвариант «без ок не стартует» (US-3.2), правка полей+старт (US-3.3), параметры (US-3.4),
   очередь (US-3.5), прогресс (US-3.6), «применить ко всем»/«позже» (US-3.7). Иначе нарушался
   **S** (Small) и **I** (Independent) — каждая теперь влезает в спринт-кусок и тестируется
   отдельно.
2. **«Обложка» (исходно одна)** → mp3-источник (US-4.1), веб-поиск (US-4.2), генерация (US-4.3),
   выбор/замена (US-4.4) — это разные подсистемы (чтение тегов / сеть / рендер / UI), каждая
   **Estimable** и **Testable** по отдельности; цепочка собирается из независимых звеньев.

Замечание по зависимостям (для беклога, не нарушает INVEST): сборка (E5) логически опирается на
распознавание (E2) и параметры (E3); это **последовательность ценности**, а не жёсткая блокировка
— каждую историю можно демонстрировать на заглушках/фикстурах.
